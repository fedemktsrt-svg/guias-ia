⚡ Fast vs Relax vs Turbo — entender el sistema de generacion

Fast mode: GPU dedicada — tu imagen se procesa en 15-60 segundos. Consume Fast GPU Hours del plan. La experiencia optima. Relax mode: Cola compartida — esperas 1-10 minutos segun la carga del servidor. Totalmente ilimitado en Standard, Pro y Mega. Para el 80% de los casos, Relax es suficiente. Turbo mode: 4x mas rapido que Fast, pero consume 2x los Fast Hours. Para iteraciones urgentes. Estrategia optima: Usa Relax para explorar y iterar · Reserva Fast para las iteraciones finales que necesitas rapido. Con Standard ($30) tienes practicamente imagenes ilimitadas via Relax.

Crea tu cuenta

3

Escribe tu primer prompt

Ajusta y genera

Selecciona la mejor de las 4 variaciones. Upscale para alta resolucion. Genera variaciones de la elegida. Descarga cuando estes satisfecho.

Capitulo 05

Parametros clave de Midjourney

Los parametros son el "codigo" de Midjourney — instrucciones especificas que van al final del prompt precedidas de "--". Dominarlos es lo que separa a un usuario basico de un usuario avanzado que tiene control real sobre sus resultados.

Parametros de Midjourney — el lenguaje avanzado para control total
--ar --style --chaos --weird --stylize --quality --no --seed --tile
--ar (Aspect Ratio) — la proporcion de tu imagen
16:9 para widescreen · 9:16 para movil · 1:1 cuadrado · 4:3 clasico · 2:3 portrait
--stylize y --chaos — creatividad vs control en el resultado
--stylize 0-1000: cuanto interpreta MJ · --chaos 0-100: variacion entre las 4 imagenes
Character Reference y Style Reference — coherencia entre imagenes
--cref para mantener el mismo personaje · --sref para consistencia de estilo visual
--ar
Aspect Ratio — proporcion de la imagen
Define las dimensiones de tu imagen. El parametro mas importante — siempre defuinelo segun el uso final. Sin --ar, Midjourney genera 1:1 por defecto.
--ar 16:9 (widescreen) · --ar 9:16 (movil/Reels) · --ar 1:1 (Instagram) · --ar 4:3 (clasico) · --ar 3:2 (foto) · --ar 2:3 (retrato) · --ar 21:9 (cine)
--stylize (--s)
Stylize — cuanto interpreta Midjourney el prompt
Valores de 0 a 1000. Bajo (0-200): muy fiel al prompt, menos creativo. Alto (750-1000): mas interpretacion artistica, mas impresionante pero menos predecible. Default: 100.
--s 0 (literal) · --s 100 (default) · --s 500 (equilibrado) · --s 1000 (maximo artistico)
--chaos (--c)
Chaos — variacion entre las 4 imagenes generadas
De 0 a 100. Bajo: las 4 imagenes son parecidas entre si. Alto: 4 interpretaciones muy diferentes del mismo prompt. Util para explorar posibilidades.
--c 0 (coherente) · --c 25 (algo de variacion) · --c 75 (variado) · --c 100 (maximo caos)
--weird (--w)
Weird — factor de rareza y originalidad
De 0 a 3000. Agrega un componente de extraneza y originalidad que no sigue esteticas convencionales. Perfecto para arte surreal, conceptual o avant-garde.
--w 250 (sutil) · --w 1000 (notablemente raro) · --w 3000 (muy experimental)
--seed
Seed — reproducibilidad y variaciones controladas
Un numero entre 0 y 4294967295. El mismo seed con el mismo prompt siempre genera el mismo resultado base. Util para variaciones controladas del mismo personaje o escena.
--seed 42 · --seed 1234567 · Obtén el seed de una imagen generada con /show o en la web
--no
Negative prompt — excluye elementos especificos
Lista lo que NO quieres en la imagen. Muy util para evitar problemas comunes como manos mal generadas, textos, fondos especificos o elementos que aparecen sin pedirlos.
--no hands, text, watermark, blurry · --no ugly, deformed, unrealistic
--tile
Tile — patron repetible y sin costuras
Genera imagenes que se repiten sin costuras visibles — perfectas para fondos, texturas, telas, papel tapiz y cualquier diseno que se tileable.
--tile (sin valor adicional) · Ideal para: texturas, patrones, fondos de pantalla
--cref / --sref
Character Ref / Style Ref — coherencia visual
--cref [URL]: mantiene el mismo personaje entre imagenes. --sref [URL]: aplica el estilo visual de una imagen de referencia. Revolucionario para proyectos con personajes recurrentes.
--cref [URL imagen personaje] --cw 100 · --sref [URL estilo] --sw 100
🔢 Multi-prompts con :: — asigna pesos a diferentes elementos

Los :: permiten dar mas o menos importancia a diferentes partes del prompt. Ejemplo: "epic mountain landscape ::2 dramatic storm clouds ::1 lone hiker ::3" — la cascada de ponderacion hace que el caminante solitario sea el elemento mas importante (::3), la montaña el segundo (::2) y las nubes el menos importante (::1). Puedes usar valores negativos (::−1) para suprimir elementos especificos dentro del prompt. Esta tecnica da control granular que no es posible con prompts de texto lineales.

Vary Region — edita solo la zona que quieres cambiar

Selecciona con el pincel la region especifica que quieres regenerar — la cara, el fondo, un objeto — y describe el cambio. El resto de la imagen permanece intacto. Es el equivalente de Photoshop Generative Fill dentro de Midjourney.

Outpainting — expande tu imagen mas alla del encuadre

Extiende la imagen en cualquier direccion — izquierda, derecha, arriba, abajo. Midjourney genera el contenido nuevo manteniendo coherencia visual con la imagen original. Perfecto para crear versiones apaisadas de imagenes verticales o viceversa.

Character Reference (--cref) — personajes consistentes

La funcion mas revolucionaria de 2024-2025: mantiene el mismo personaje coherente entre multiples imagenes con solo pasar una URL de referencia. Para libros ilustrados, campanas de marketing, personajes de marca y cualquier proyecto con un personaje recurrente.

Image Prompts — usa imagenes como referencia visual

Pega una URL de imagen al principio del prompt y Midjourney la usa como referencia visual ademas del texto. Combina hasta 5 imagenes de referencia. Controla el peso de cada imagen con --iw (image weight). Util para composiciones y estilos visuales especificos.

Blend — funde dos imagenes en una

Combina 2-5 imagenes para crear una nueva que mezcla los elementos de todas. El /blend de Discord o la funcion en la web web. Util para crear estilos hibridos, texturas mezcladas y conceptos visuales que combinan referencias distintas.

Capitulo 09

Errores comunes (y como evitarlos)

Los errores mas frecuentes de nuevos usuarios en Midjourney — la mayoria relacionados con prompts mal estructurados, parametros mal usados y expectativas incorrectas sobre lo que la herramienta puede y no puede hacer.

Errores comunes en Midjourney — por que tus imagenes no salen como quieres
Los habitos de prompting que generan resultados mediocres y como corregirlos
Prompts demasiado vagos — "a beautiful woman" no es un prompt, es un susurro
Midjourney necesita especificidad: sujeto + contexto + estilo + iluminacion + ratio
No usar --ar para el formato correcto antes de generar
Siempre define la proporcion segun el uso final: social, web, impresion, video
📝
Prompts demasiado cortos y vagos — la causa numero uno de resultados mediocres
"A beautiful woman" o "a mountain at sunset" son prompts que producen imagenes genericas. Midjourney tiene que adivinar demasiado cuando el prompt es vago — y sus elecciones por defecto son estilos intermedios y poco personales. La cantidad de detalle en el prompt es directamente proporcional a la calidad del resultado.
✅ Estructura minima: sujeto principal + entorno/contexto + estilo visual + iluminacion + composicion + calidad + parametros (--ar, --stylize, --v). Un prompt de 40-80 palabras bien estructurado produce resultados infinitamente mejores que uno de 5.
📐
No usar --ar y generar siempre en 1:1
Sin --ar, Midjourney genera imagenes cuadradas 1:1 por defecto. Para Instagram esta bien, pero para thumbnails de YouTube necesitas 16:9, para Reels 9:16, para portadas de libros 2:3, para wallpapers de pantalla panoramica 21:9. Generar en el ratio incorrecto y recortar despues pierde resolucion y composicion.
✅ Define --ar como lo primero despues del prompt: --ar 16:9 para horizontal, --ar 9:16 para vertical movil, --ar 1:1 cuadrado, --ar 4:5 para Instagram portrait, --ar 2:3 para libros y portadas. El ratio correcto cambia completamente la composicion generada.
🔢
Ignorar --stylize y siempre usar el valor por defecto
--stylize 100 es el valor por defecto — un equilibrio que a menudo no es ni suficientemente literal ni suficientemente artistico. Para fotografia realista, baja a --s 50-200. Para arte conceptual impresionante, sube a --s 750-1000. Muchos usuarios nunca cambian este parametro y se quedan con resultados mediocres.
✅ Para fotografia e ilustracion realista: --s 50-300. Para arte conceptual y resultados impactantes: --s 600-900. Para maxima interpretacion artistica: --s 1000. Experimenta con el mismo prompt en --s 100 y --s 800 — la diferencia es radical.
🌐
Escribir prompts en espanol
Midjourney entiende espanol pero fue entrenado principalmente con contenido en ingles — especialmente el vocabulario fotografico y artistico. "Golden hour", "bokeh", "Rembrandt lighting", "hyperrealistic" son terminos que el modelo interpreta con mucha mas precision en ingles que en espanol. La diferencia en calidad es notable.
✅ Escribe siempre en ingles. Si no conoces los terminos tecnicos fotograficos o artisticos en ingles, usa ChatGPT o DeepL para traducir tu descripcion y luego refina el vocabulario. El resultado mejora significativamente.
👐
No usar --no para los problemas comunes de Midjourney
Midjourney tiene problemas conocidos: manos con dedos de mas o mal formadas, texto ilegible en las imagenes, marcas de agua invisibles y fondos que no quieres. Sin usar --no para excluirlos explicitamente, aparecen frecuentemente — especialmente en imagenes de personas.
✅ Para retratos y personas: agrega --no deformed hands, extra fingers, watermark, signature al final del prompt. Para imagenes de producto: --no shadow, text, background elements. Para arte abstracto que no quieres que sea figurativo: --no people, faces, hands.
💰
Desperdiciar Fast Hours en variaciones que podrian hacerse en Relax
El error financiero mas comun: usar Fast mode para toda la fase de exploracion y quedarse sin Fast Hours antes de refinar los mejores resultados. Fast es para cuando necesitas velocidad. Relax es ilimitado en Standard y superior — y la espera de 2-5 minutos es perfectamente aceptable para la fase de iteracion.
✅ Flujo optimo: usa Relax para el 80% del trabajo (exploracion, variaciones, iteraciones). Cambia a Fast solo para las iteraciones finales que necesitas rapidamente o cuando tienes un deadline. Guarda Turbo para emergencias. Asi los 15 Fast Hours de Standard duran todo el mes.
