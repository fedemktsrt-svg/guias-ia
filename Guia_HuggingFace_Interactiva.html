Capítulo 01

¿Qué es Hugging Face?

Hugging Face es la plataforma de referencia mundial para la comunidad de inteligencia artificial. Funciona como el "GitHub de la IA": un lugar donde investigadores, empresas y desarrolladores comparten, descubren y usan modelos de IA de forma abierta y colaborativa.

🧠

Hub de modelos

Más de 1 millón de modelos de IA listos para usar: texto, imagen, audio, video, código y más. Todos descargables y muchos ejecutables gratis en el navegador.

🚀

Spaces — Apps de IA

Más de 300.000 aplicaciones de IA interactivas creadas por la comunidad. Probá cualquier modelo sin escribir código, directamente en el navegador.

📦

Datasets

Medio millón de datasets para entrenar modelos de IA. Desde texto hasta imágenes, audio y datos multimodales, clasificados por tarea y dominio.

📚

Transformers Library

La biblioteca Python más popular del mundo de IA. Con pocas líneas de código tenés acceso a miles de modelos para cualquier tarea de procesamiento de IA.

⚡

Inference API

API gratuita para usar modelos de IA en tus aplicaciones. Sin configurar servidores ni infraestructura. Listo para producción con un simple HTTP request.

🏢

Organizaciones

Meta, Google, Microsoft, Mistral, Stability AI y miles de startups publican sus modelos en Hugging Face. Es el estándar de la industria para compartir IA.

Hugging Face democratizó el acceso a la inteligencia artificial de la misma forma que GitHub democratizó el código abierto. Cualquier persona en el mundo puede hoy acceder a los mismos modelos que usan las grandes empresas tecnológicas.

Capítulo 02

Cómo acceder y crear tu cuenta

Acceder a Hugging Face es completamente gratuito. La gran mayoría del contenido es público y no requiere cuenta, aunque registrarte te da acceso a funciones adicionales.

Paso 1
Visitá huggingface.co
Entrá a huggingface.co desde cualquier navegador. Podés explorar modelos, Spaces y datasets sin registrarte. La experiencia sin cuenta es muy completa.
Paso 2
Creá tu cuenta gratuita
Hacé clic en "Sign Up" y registrate con tu email o con GitHub/Google. Tu nombre de usuario será tu identidad pública en la plataforma (como en GitHub).
Paso 3
Configurá tu perfil
Completá tu perfil con tu nombre, foto y una descripción. Podés indicar si sos investigador, desarrollador, estudiante o emprendedor. Esto ayuda a la comunidad a conectarse.
Paso 4
Obtené tu Access Token
En Settings → Access Tokens generá tu token de API gratuito. Lo vas a necesitar para usar modelos desde código Python o para acceder a la Inference API desde tus aplicaciones.
Paso 5 (opcional)
Pedí acceso a modelos gated
Algunos modelos premium (como Llama de Meta o Gemma de Google) requieren aceptar términos de uso. Buscá el modelo, completá el formulario y generalmente el acceso se aprueba en minutos.
🤗

¿Para qué sirve la cuenta en Hugging Face?

Con cuenta podés: usar la Inference API, guardar modelos favoritos, crear tus propios Spaces y repositorios, unirte a organizaciones, participar en la comunidad y acceder a modelos gated. El plan gratuito es muy generoso.

Capítulo 04 — Sin código

Spaces: aplicaciones de IA listas para usar

Los Spaces son la forma más rápida de probar y usar modelos de IA sin escribir código. Son aplicaciones web interactivas creadas por la comunidad que corren modelos de IA directamente en el navegador.

¿Qué son los Spaces?

Cada Space es una mini-aplicación web que expone un modelo de IA con una interfaz gráfica. Podés subir imágenes, escribir texto, grabar audio y ver los resultados al instante, todo sin instalar nada.

  • Más de 300.000 apps de IA disponibles gratis
  • Creados con Gradio o Streamlit, frameworks de Python
  • Muchos corren en CPU gratuita, otros requieren GPU
  • Podés duplicar cualquier Space y personalizarlo
  • Algunos están integrados directamente en páginas de modelos

Capítulo 06

Inference API: IA en tu app sin servidores

La Inference API de Hugging Face te permite usar modelos de IA directamente desde tu aplicación con una simple llamada HTTP. Sin configurar servidores, sin instalar dependencias. Solo tu token y un endpoint.

¿Cómo funciona la Inference API?

La Inference API es un servicio gratuito (con límites de uso) que te permite hacer requests HTTP a cualquier modelo del Hub de Hugging Face y recibir la respuesta procesada por el modelo directamente.

  • Gratis para modelos públicos con tu cuenta de HF
  • Funciona con cualquier lenguaje de programación
  • Soporta texto, imagen, audio y más tipos de input
  • Rate limits generosos para proyectos personales
  • Serverless Inference para escalar en producción

Ejemplo: Usar un LLM con Python

from huggingface_hub import InferenceClient # Inicializá el cliente con tu token client = InferenceClient( model="meta-llama/Llama-3.3-70B-Instruct", token="hf_tu_token_aqui" ) # Enviar un mensaje y obtener respuesta response = client.chat_completion( messages=[ {"role": "user", "content": "Explicame qué es machine learning en 3 líneas"} ], max_tokens=500 ) print(response.choices[0].message.content)

Ejemplo: Generar imagen con FLUX

from huggingface_hub import InferenceClient client = InferenceClient(token="hf_tu_token_aqui") # Generar imagen con FLUX.1 image = client.text_to_image( prompt="A professional portrait of a latina entrepreneur, warm lighting, photorealistic", model="black-forest-labs/FLUX.1-dev" ) # Guardar la imagen image.save("mi_imagen.png") print("Imagen generada y guardada")

Ejemplo: Transcripción de audio con Whisper

from huggingface_hub import InferenceClient client = InferenceClient(token="hf_tu_token_aqui") # Transcribir un archivo de audio transcription = client.automatic_speech_recognition( audio="mi_audio.mp3", model="openai/whisper-large-v3" ) print(f"Transcripción: {transcription.text}")
Paso 1: Andá a huggingface.co → tu avatar → Settings → Access Tokens

Paso 2: Hacé clic en "New Token", elegí el nombre (ej: "mi-proyecto") y el rol. Para usar modelos elegí "Read", para subir modelos elegí "Write".

Paso 3: Copiá el token (empieza con "hf_") y guardalo en un lugar seguro. Solo se muestra una vez.

Buenas prácticas: Nunca hardcodees el token en tu código. Usá variables de entorno: os.environ['HF_TOKEN'] o un archivo .env con python-dotenv.
Instalar el Hub client:
pip install huggingface_hub

Instalar Transformers (para más control):
pip install transformers torch

Instalar Diffusers (para imágenes):
pip install diffusers transformers accelerate

Login desde terminal:
huggingface-cli login
¿Qué son? Los Inference Endpoints son una forma de deployar cualquier modelo del Hub en tu propia instancia dedicada de nube (AWS, GCP, Azure) con un click.

¿Cuándo usarlos? Cuando la Inference API gratuita no es suficiente por los rate limits, cuando necesitás latencia garantizada, o cuando tenés modelos privados que no podés compartir públicamente.

Precio: Pagás por el tiempo que el endpoint está activo. Podés configurarlos para que se "apaguen" cuando no hay tráfico (scale to zero) para minimizar costos.

Configuración: huggingface.co/inference-endpoints → New Endpoint → elegí el modelo, la región y el tipo de instancia. En 2-3 minutos tenés tu endpoint listo.

Capítulo 08

30 Usos prácticos listos para implementar

Casos de uso concretos con el modelo recomendado para cada tarea en Hugging Face. Cada uno incluye el modelo a usar y el tipo de input/output esperado.

01

Análisis de sentimiento de reseñas

Texto
Clasificá automáticamente si una reseña de cliente es positiva, negativa o neutral. Ideal para monitoreo de marca y feedback de productos.
Modelo: nlptown/bert-base-multilingual-uncased-sentiment Input: Texto de la reseña en español Output: Puntuación de 1 a 5 estrellas Ejemplo de uso: "Este producto es increíble, superó mis expectativas" → Positivo (5 estrellas)
02

Resumen automático de documentos

Texto
Resumí artículos largos, contratos, reportes o correos en un párrafo clave. Ahorrá horas de lectura con resúmenes precisos y concisos.
Modelo: facebook/bart-large-cnn Input: Texto largo (artículo, documento) Output: Resumen de 1-3 párrafos Parámetros clave: - max_length: longitud máxima del resumen - min_length: longitud mínima - do_sample: False para mayor coherencia
03

Clasificación de emails/tickets

Texto
Clasificá automáticamente emails de soporte en categorías: facturación, soporte técnico, ventas, reclamo. Sin necesidad de entrenamiento previo.
Modelo: facebook/bart-large-mnli (zero-shot) Input: Texto del email + lista de categorías Output: Categoría más probable con score de confianza Ventaja: Funciona sin datos de entrenamiento. Solo definís las categorías en texto plano.
04

Detección de idioma automática

Texto
Identificá automáticamente el idioma de cualquier texto. Útil para rutear mensajes al equipo correcto o para filtrar contenido por idioma.
Modelo: papluca/xlm-roberta-base-language-detection Input: Cualquier texto Output: Código de idioma (es, en, pt, fr...) + probabilidad Soporta más de 100 idiomas. Precisión >99% para textos de más de 20 palabras.
05

Extracción de información (NER)

Texto
Extraé automáticamente nombres de personas, organizaciones, fechas, lugares y montos de textos no estructurados como contratos o noticias.
Modelo: dslim/bert-base-NER Input: Texto con entidades Output: Lista de entidades etiquetadas Ejemplo: "Juan García firmó el 15 de enero con Acme Corp" → PER: Juan García | DATE: 15 enero | ORG: Acme Corp

Texto

Chatbot de soporte con RAG

Negocio
Creá un chatbot que responde preguntas basándose en tu documentación interna usando RAG (Retrieval Augmented Generation). Sin alucinaciones, respuestas basadas en hechos.
Stack recomendado: - Embeddings: sentence-transformers/all-MiniLM-L6-v2 - LLM: mistralai/Mistral-7B-Instruct-v0.3 - Vector DB: Faiss o ChromaDB (gratuitos) Flujo: documento → embeddings → base vectorial → pregunta → recuperar contexto → LLM responde Ventaja: respuestas 100% basadas en TUS datos.

Búsqueda semántica en documentos

Negocio
Buscá por significado, no solo por palabras clave. Encontrá el pasaje relevante en miles de documentos sin necesidad de coincidencia exacta de palabras.
Modelo: sentence-transformers/all-MiniLM-L6-v2 Flujo: 1. Convertí cada documento a embeddings (vectores) 2. Almacenalos en Faiss o Pinecone 3. Cuando el usuario busca, convertí su consulta a vector 4. Encontrá los documentos más similares por distancia Ventaja: "¿Cómo cancelo mi suscripción?" encuentra "Para dar de baja tu cuenta..." sin coincidencia exacta.
22

Fine-tuning de modelos propios

Dev
Ajustá un modelo preentrenado con tus propios datos para que se especialice en tu dominio específico. Mucho más barato que entrenar desde cero.
Librería: transformers + peft (para LoRA) Proceso: 1. pip install transformers peft datasets 2. Preparar dataset en formato JSONL 3. Definir modelo base (ej: Llama 3 8B) 4. Configurar LoRA adapters 5. Entrenar con Trainer o SFTTrainer 6. Subir el modelo fine-tuned al Hub Con LoRA podés fine-tunear en una GPU consumer.
23

Correr modelos localmente con Transformers

Dev
Descargá y corré cualquier modelo de HF directamente en tu computadora. Sin internet después de la descarga, sin costos de API, con privacidad total.
from transformers import pipeline # Pipeline de generación de texto generator = pipeline("text-generation", model="mistralai/Mistral-7B-Instruct-v0.3", device_map="auto") # usa GPU si está disponible result = generator("¿Qué es la IA?", max_new_tokens=200) print(result[0]['generated_text']) # Se descarga la primera vez (~14GB para 7B) # Después corre offline completamente.

Capítulo 09

Errores comunes al usar Hugging Face

Evitá los obstáculos que frenan a la mayoría de principiantes cuando empiezan a explorar el hub de IA más grande del mundo.

El error: Ver Hugging Face solo como un lugar para chatear con modelos de lenguaje, ignorando el 95% de lo que ofrece la plataforma.

La realidad: Hugging Face es una plataforma para toda la IA: texto, imagen, audio, video, código, ciencia, bioinformática y más. Es el lugar donde viven los modelos que luego usan servicios como ChatGPT, Midjourney y otros. La mayoría de las herramientas de IA que pagás mensualmente usan modelos que podés acceder gratis en HF.
El error: Intentar correr un modelo de 70B parámetros directamente en tu laptop y que tarde horas o se quede sin memoria RAM.

La solución: Para uso local sin GPU usá modelos de 7B o menos en formato GGUF (cuantizados). Para modelos más grandes usá la Inference API gratuita de HF que corre en servidores de HF, o Google Colab que te da GPU gratuita. Regla general: 1B parámetros ≈ 2GB de RAM para el modelo completo, 0.5GB con Q4 quantización.
El error: Escribir el token directamente en el código Python y luego subirlo a GitHub público, exponiendo tu cuenta de HF.

La solución: Siempre usá variables de entorno. Creá un archivo .env con HF_TOKEN=hf_tu_token, agregalo al .gitignore y leelo en Python con os.getenv('HF_TOKEN'). O usá huggingface-cli login que guarda el token de forma segura en tu sistema.
El error: Asumir que el modelo más grande es siempre el mejor. Usar Llama 70B para clasificar si un texto es positivo o negativo cuando un BERT de 110M funciona igual o mejor y es 100x más rápido.

La solución: Elegí el modelo según la tarea. Para clasificación, NER y tareas específicas: modelos pequeños fine-tuneados (BERT, RoBERTa). Para generación de texto y razonamiento: modelos más grandes. Para producción: priorizá velocidad y costo sobre tamaño máximo.
El error: Usar un modelo en producción comercial sin revisar su licencia, asumiendo que "código abierto" significa "uso libre para todo".

La solución: Siempre revisá la licencia en la página del modelo. Las más comunes son MIT y Apache 2.0 (uso comercial libre), Llama Community License (uso libre hasta cierta escala), y CC-BY-NC (solo uso no comercial). Para proyectos comerciales, filtrá en el Hub por licencias comerciales compatibles.
