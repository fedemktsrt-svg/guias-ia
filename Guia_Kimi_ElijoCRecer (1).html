Cómo escribir buenos prompts para Kimi

Kimi tiene características únicas que permiten estrategias de prompting especiales, sobre todo para aprovechar su contexto masivo.

Prompt básico vs. prompt para Kimi

❌

Prompt básico

"Resume este documento" → Resultado genérico y superficial, Kimi puede procesar solo parte.

✅

Prompt optimizado para Kimi

"Lee COMPLETAMENTE el documento adjunto. Eres un analista senior. Crea: 1) Resumen ejecutivo 200 palabras, 2) 5 hallazgos críticos, 3) Riesgos identificados."

La fórmula de 6 elementos para Kimi

👤

1. Rol

"Actúa como analista financiero senior..." — Da a Kimi el marco de experto correcto para tu tarea.

📄

2. Material

"He adjuntado [N documentos]. Lee TODO antes de responder." — Instrucción crítica para el contexto largo.

✅

3. Tarea

Verbos específicos: analiza, identifica, compara, extrae, sintetiza. Evita palabras vagas.

🔍

4. Profundidad

"Análisis profundo" vs. "resumen ejecutivo". Kimi ajusta la profundidad según la instrucción.

📐

5. Formato

Tabla, lista numerada, párrafos, JSON, Markdown. Kimi respeta con precisión el formato pedido.

🚫

6. Restricciones

"No incluyas recomendaciones de inversión", "Máximo 300 palabras". Las restricciones acotan y mejoran.

¿Qué es? Cargar grandes cantidades de información antes de hacer preguntas.

Cómo hacerlo:
1. Adjunta todos los documentos relevantes de una vez
2. Incluye: "Procesa TODO el material. Confirma cuando hayas terminado."
3. Espera la confirmación de Kimi
4. Luego haz todas tus preguntas sobre el material

Por qué funciona: Kimi puede mantener 2M de caracteres en memoria activa, así que puedes hacer múltiples preguntas relacionando información de distintas fuentes sin recargar nada.
Usa Instant para: Preguntas directas, traducciones, reformulaciones, conversación casual, resúmenes simples.

Usa Thinking para: Matemáticas complejas, lógica multi-paso, análisis de argumentos, código difícil, problemas que requieren verificación cruzada.

Activa Thinking incluyendo: "Piensa paso a paso antes de responder" o "Usa tu modo de razonamiento extendido".
Kimi es excelente en español y excepcional en chino. A diferencia de otros modelos, no necesitas escribir en inglés para obtener buenos resultados en Kimi.

Sin embargo, para tareas muy técnicas de programación, usar términos técnicos en inglés (como nombres de funciones o frameworks) dentro de un prompt en español produce resultados más precisos.

Recomendación: Escribe en el idioma con el que te sientas más cómodo. Kimi lo entenderá bien.

Capítulo 09

Errores comunes y cómo evitarlos

Kimi tiene particularidades únicas que generan errores igualmente únicos. Conocerlos te ahorrará frustración y créditos desperdiciados.

Errores específicos de Kimi

❌

No decirle que lea TODO el documento

Sin instrucción explícita, Kimi puede procesar solo una parte. Siempre incluye: "Lee y procesa COMPLETAMENTE todos los archivos antes de responder."

❌

Usar Instant para tareas que requieren Thinking

Matemáticas, código complejo y análisis multi-paso en modo Instant producen respuestas superficiales. Activa Thinking para estas tareas.

❌

Instrucciones vagas para OK Computer

"Haz algo sobre marketing" → El agente no sabe qué hacer. Especifica: tipo de entregable, audiencia, objetivo y formato esperado.

⚠️

Cada sesión es independiente

Kimi no recuerda conversaciones anteriores. Para proyectos continuos, crea un documento de contexto y cópialo al inicio de cada nueva sesión.

🔒

Privacidad: Kimi es un servicio de empresa china

Las conversaciones se almacenan en servidores de Moonshot AI. Evita compartir secretos comerciales muy sensibles, datos personales críticos o información confidencial de clientes sin evaluar primero las políticas de privacidad vigentes.
